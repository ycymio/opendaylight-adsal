Groupings

  • common-counters

    • Description: Counters that exist both per session, and also globally, accumulated from all sessions.
      • leaf: in-rpcs
        • description: Number of correct messages received.
        • type: (urn:ietf:params:xml:ns:yang:ietf-yang-types)zero-based-counter32
      • leaf: in-bad-rpcs
        • description: Number of messages received when an message was expected, that were not correct messages. This includes XML parse errors and errors on the rpc layer.
        • type: (urn:ietf:params:xml:ns:yang:ietf-yang-types)zero-based-counter32
      • leaf: out-rpc-errors
        • description: Number of messages sent that contained an element.
        • type: (urn:ietf:params:xml:ns:yang:ietf-yang-types)zero-based-counter32
      • leaf: out-notifications
        • description: Number of messages sent.
        • type: (urn:ietf:params:xml:ns:yang:ietf-yang-types)zero-based-counter32

RPC Definitions

get-schema

  • Description: This operation is used to retrieve a schema from the NETCONF server. Positive Response: The NETCONF server returns the requested schema. Negative Response: If requested schema does not exist, the is 'invalid-value'. If more than one schema matches the requested parameters, the is 'operation-failed', and is 'data-not-unique'.
    • container: input
        • leaf: identifier
          • description: Identifier for the schema list entry.
          • type: (urn:ietf:params:xml:ns:yang:1)string
        • leaf: version
          • description: Version of the schema requested. If this parameter is not present, and more than one version of the schema exists on the server, a 'data-not-unique' error is returned, as described above.
          • type: (urn:ietf:params:xml:ns:yang:1)string
        • leaf: format
          • description: The data modeling language of the schema. If this parameter is not present, and more than one formats of the schema exists on the server, a 'data-not-unique' error is returned, as described above.
          • type: (urn:ietf:params:xml:ns:yang:1)identityref
    • container: output
        • anyxml: data
          • description: Contains the schema content.
